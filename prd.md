# [PRD] 감성 3D 뽀모도로 타이머: 봄이 오는 시간 (Cherry Blossom Timer)

## 1. 제품 개요 (Product Overview)
본 서비스는 사용자의 집중과 휴식 주기를 벚꽃 나무의 생애 주기에 투영하여 시각적 즐거움과 정서적 안정감을 제공하는 **웹 기반 3D 뽀모도로 집중타이머** 애플리케이션입니다.

* **프로젝트명:** 봄이 오는 시간
* **실행 환경:** 웹 브라우저 (데스크톱/모바일 대응)
* **렌더링 방식:** Blender로 제작한 3D 벚꽃나무 모델을 웹 3D 엔진으로 실시간 렌더링.
* **제작 현황:** Blender 3D 모델 미제작 (예정). 본 PRD는 제작 전 기준 명세이며, Blender MCP를 활용해 모델링 진행 예정.

## 2. 핵심 컨셉 (Core Concept)
* **집중 (45분):** 만개한 나무에서 꽃잎이 비처럼 내리며 바닥에 쌓이는 과정 (낙화)
* **휴식 (15분):** 비어버린 가지에서 다시 꽃봉오리가 맺히고 꽃이 피어나는 과정 (개화)

---

## 3. 주요 기능 요구사항 (Functional Requirements)

### 3.1. 타이머 로직 및 모드
* **고정 주기:** 집중 45분 / 휴식 15분 (기본 설정).
* **상태 제어:** * `Start`: 타이머 시작 및 꽃잎 애니메이션 활성화.
    * `Pause`: 타이머 및 애니메이션 일시 정지.
    * `Reset`: 초기 상태(만개한 나무)로 복구.
* **알림 시스템:** * 45분 종료 시: "낙화가 완료되었습니다. 잠시 쉬어가세요." 알람.
    * 15분 종료 시: "다시 꽃이 만개했습니다. 집중할 시간입니다." 알람.

### 3.2. 시각적 연출 (3D Visual Animation)
* **집중 모드 (45분간의 낙화):**
    * 나무 가지에 부착된 꽃(인스턴스)의 개수/밀도가 $T=0$에서 $T=45$분까지 선형적으로 감소.
    * 꽃잎 파티클이 3D 공간에서 바람 벡터에 따라 흩날리며 바닥으로 하강 (중력 + 난류 시뮬레이션).
    * 바닥 평면에 쌓인 꽃잎(파티클 누적 또는 핑크 텍스처 밀도)이 서서히 증가.
* **휴식 모드 (15분간의 개화):**
    * 가지 끝에 꽃봉오리 메쉬가 생성·스케일업되며 나무의 볼륨감이 회복됨.
    * 바닥에 쌓인 꽃잎은 불투명도(Alpha)가 낮아지며 서서히 소멸.
* **카메라/연출:** 자동 회전(완만한 오빗) 또는 고정 시점. 진행률에 따라 라이팅 톤(따뜻함) 변화.

## 5. 3D 에셋 구성 (3D Assets — Blender)

이미지 단계 전환 방식 대신, **단일 3D 벚꽃나무 모델**의 파라미터(꽃 밀도, 파티클, 바닥 텍스처)를 진행률에 따라 실시간 제어합니다. 정적인 4단계는 참조용 상태(State)로만 정의합니다.

### 5.1. 제작 에셋 (Blender MCP로 모델링 예정)
* **트렁크/가지:** 첨부 레퍼런스 이미지(굵은 둥치 + 넓게 퍼진 수관)를 기반으로 한 벚꽃나무 본체 메쉬.
* **꽃 (Blossom):** 가지에 인스턴싱할 개별 꽃 메쉬. 핑크 그라데이션 머티리얼.
* **꽃봉오리 (Bud):** 휴식 모드 개화용 작은 봉오리 메쉬.
* **꽃잎 파티클 (Petal):** 낙화용 단일 꽃잎 로우폴리 메쉬.
* **바닥 (Ground):** 잔디 평면 + 쌓인 꽃잎 표현용 머티리얼.

### 5.2. 진행률에 따른 상태 (참조)
| 상태 | 진행률 | 꽃 밀도 | 파티클 | 바닥 |
| :--- | :--- | :--- | :--- | :--- |
| **Full** | 집중 0% / 휴식 100% | 100% | 없음 | 깨끗 |
| **Half** | 집중 50% | 50% | 활발히 낙하 | 부분 누적 |
| **Empty** | 집중 100% | 0% | 잔여 낙하 | 가득 누적 |
| **Budding** | 휴식 진행 | 봉오리 생성·확대 | 없음 | 서서히 소멸 |

### 5.3. 웹 출력 가이드
* GLB 단일 파일로 익스포트, Draco 압축 적용.
* 폴리곤 수는 모바일 대응 위해 최적화(로우폴리 + 노멀맵 권장).
* 꽃/꽃잎은 개별 메쉬로 분리 익스포트하여 런타임 인스턴싱.

## 6. UI/UX 디자인 가이드
* **씬(Scene):** 3D 벚꽃나무가 화면 전체를 채우는 풀스크린 캔버스. 하늘 그라데이션/환경광으로 분위기 연출.
* **타이머 표시:** 3D 캔버스 위에 HTML/CSS 오버레이(HUD)로 배치. 가독성을 해치지 않는 반투명 폰트로 중앙 또는 하단.
* **컨트롤:** Start / Pause / Reset 버튼을 Glassmorphism 스타일 HUD로 배치.
* **인터랙션:** 버튼 클릭 시 부드러운 페이드 인/아웃, 상태 전환 시 카메라/라이팅 트랜지션.

## 7. Blender 제작 로드맵 (Production Roadmap)

> Blender MCP(`mcp__blender__*`)를 통해 단계적으로 모델링 진행 예정.

1. **씬 초기화** → verify: 빈 씬 + 기본 라이팅/카메라 셋업 확인.
2. **트렁크/가지 모델링** → verify: 레퍼런스 이미지와 유사한 수형(둥치+수관) 확인.
3. **꽃·꽃봉오리·꽃잎 에셋 제작** → verify: 개별 메쉬 + 핑크 머티리얼 적용 확인.
4. **가지에 꽃 인스턴싱** → verify: 만개(Full) 상태 뷰포트 렌더 확인.
5. **GLB 익스포트(Draco)** → verify: 웹에서 R3F 로드 및 폴리곤/용량 확인.

## 8. 기대 효과 (Expected Outcomes)
* 시각적 보상(꽃이 피는 과정)을 통해 휴식의 질 향상.
* 시간의 흐름을 '꽃잎이 떨어지는 양'으로 직관적으로 파악하여 집중도 유지.
* 단순한 도구가 아닌 감성적인 데스크테리어(Deskterior) 요소로 활용.
